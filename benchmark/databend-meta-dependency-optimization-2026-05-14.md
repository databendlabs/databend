# Databend Meta Dependency Optimization Benchmark

Date: 2026-05-14

## Scope

This benchmark compares Databend `ups/main` with the patched Databend branch that uses the released optimized Databend Meta dependency tags.

The benchmark target is Databend Meta in a 2-node cluster. The measured workload is `upsert_kv`.

## Dependency Optimization

The patched branch switches Databend to released Databend Meta tags that include the `raft-log` WAL flush optimizations:

- bounded group-commit wait for WAL flushes;
- configurable flush batch size;
- vectored batch writes;
- WAL flush latency percentiles exposed through raft-log stats;
- richer meta service logging for raft-log and Openraft runtime stats.

The patched code state benchmarked here is:


This report is amended into that dependency-update commit. The report-only amend changes the commit object, but does not change the benchmarked code or dependency resolution.

## Interpretation

The patched dependency stack improves throughput by about 10% to 13% in these single runs. Average client-side latency also drops by about 10% to 12%.

The improvement is consistent with the raft-log batching change: at 64 and 256 clients the WAL flush worker batches many logical writes into one sync batch, with average batch sizes of 19 and 54. This reduces fsync amortized cost per write. With 4 clients the average batch size is still 1, so the improvement there should not be attributed only to group commit.

The core persistence path is still expensive. In the patched runs, `submitted->persisted` remains around 16ms to 20ms at p90 and 21ms to 25ms at p99. The new group commit mainly amortizes this cost across more writes; it does not remove the underlying sync latency.

The 64-client run has a mixed latency signal: throughput and average latency improve, but p99 and slow-RPC/slow-IO counts are worse than `ups/main`. This looks like tail variance around WAL sync and should be confirmed with repeated runs before treating the p99 regression as stable.
